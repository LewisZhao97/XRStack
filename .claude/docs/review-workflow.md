# Review Workflow

1. Code changes require review by `lead-programmer` (or relevant specialist lead)
2. XR interaction changes require review by `xr-specialist`
3. SDK public API changes require review by `lead-programmer` and `technical-director`
4. Architecture changes require sign-off from `technical-director`
5. Cross-system changes require coordination through the user
6. Security-sensitive changes require careful review by `lead-programmer` and `technical-director`
